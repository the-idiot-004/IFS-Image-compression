Fractal IFS Image Compression

Iterated Function System · encode a target, iterate from any starting image to its attractor

∞ How Fractal IFS Compression Works ▾
▲

Contractive Maps

An IFS is a finite set of contractive affine transformations. "Contractive" means each map shrinks distances in image space — guaranteed here by 2× spatial downsampling of domain blocks. Banach's Fixed Point Theorem then guarantees convergence from any starting image.

▦

Encoding

The target is tiled into small range blocks (e.g. 4×4 px). For each we search a pool of domain blocks — patches twice as large, averaged down to match size. We store only: which domain block + brightness scale s and offset o — not the pixels themselves.

◎

The Fixed-Point Attractor

The stored maps define a unique fixed point A in image space such that W(A) = A. Applying the IFS operator to the attractor reproduces it exactly. This attractor closely approximates the original target — that is the "compressed" image.

↻

Iteration / Decoding

Start from any image x₀ — even a photo of Al Pacino. Apply transforms: x₁ = W(x₀), x₂ = W(x₁), … After a handful of iterations the sequence converges to the attractor, morphing entirely into the target (e.g. Monica Bellucci). No target pixels are stored.

Based on Jacquin (1992) · 2× downsampling ensures |W(x)−W(y)| ≤ c|x−y| with c < 1 for all images x, y · Banach's theorem gives unique convergence
